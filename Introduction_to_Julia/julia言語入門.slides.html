pythonとの違いは?

基本的な文法はほとんどpythonと同じです。
関数はmatlabと似たものが多かったのですが、現在は多少変わりつつあります。

主な違いはNoteworthy Differences...に良くまとまっています。

  • 配列のインデックスは "1" から始まる。
  • 配列の最後の要素の指定は"-1" でなく"end"。
  • "Comprehensions"はまだ"for" のみ利用可能?
  • juliaの"配列"は"列(Column)"がメインです。
  • "return"がなくても、最後に評価されたものが帰ってきます。
  • forなどにスコープがあります
  • end文がつきます rubyと同じです

などがあります。

Task(コルーチン)

コルーチンとは,処理をいったん中断し,その後必要に応じてにその処理の続きから再開することができる制御構造です

  • produce
    処理を中断する(指定する)

  • Task
    タスクを生成する

  • consum
    タスクを再開する

コルーチンを使ってギブスサンプラーを作ってみよう!!

  • 多変量正規分布からサンプリング
  • 入力ベクトル${\bf x}=[x, y]^T$,平均ベクトルを$\mu$、共分散行列を$\Sigma$とすると多変量正規分布は

$$ p(x, y) = \frac{1}{(\sqrt{2\pi})^2{\rm det}(\Sigma)} \exp\left( - \frac{({\bf x}-\mu)^T\Sigma^{-1}({\bf x}-\mu)}{2} \right)$$

In [26]: 
# 初回実行時はPkg.add("Distributions")で
# Distributionsパッケージをインストールしておく
using Distributions

#平均[0, 0], 分散[1 ρ; ρ 1] の場合
ρ = 0.90
μ = [0, 0]
S  = [1 ρ; ρ 1]
Sⁱ = inv(S)
detS = det(S)

#目的関数
f(x) = exp(-(x-μ)'Sⁱ*(x-μ)/2)/(2π*sqrt(detS))
Out[26]:
f (generic function with 1 method)
  • ギブスサンプリング
    • 同時分布においてサンプルしたい確率変数を除いた確率変数に条件付けされた分布からのサンプルを繰り返すことで目的とする分布からのサンプルを得る手法
    • xのサンプルはp(x | y)から、yのサンプルはp(y | x)からサンプリング
    • 平均ベクトル$\mu=[0, 0]^T$、共分散行列の対角項が1, 共分散項が$\rho$のとき$p(x|y)=N(\rho y, 1 - \rho^2)$
In [27]: 
#条件付き分布から一つサンプリング
function sample_one(x)
    new_x = rand(Normal(ρ*x[2], 1-ρ^2))
    new_y = rand(Normal(ρ*new_x, 1-ρ^2))
    return [new_x, new_y]
end

#コルーチン
function sampler(burnin)
  x = zeros(2)
  # BURNINの期間はひたすらサンプルを繰り返すのみで値は返さない
  for i = 1:burnin
    x = sample_one(x)
  end
  while true
    x = sample_one(x)
    produce(x)  #produce
  end
end

マクロを搭載

  • @から始まるものは、マクロと呼ばれるものです。
  • ストリングスタイプのマクロも存在
    • 正規表現リテラルのように使える r" "もマクロ
      • match(r"^\s*(?:#|$)", "# a comment")

juliaにおけるコードの評価

  • 以上のようにExpressonを引数に持ち返り値もExpressonであるような関数を定義してもいいのではないかと思うかもしれない
  • しかしマクロを使うメリットはコードの変換・生成にある
    • juliaにおけるコード評価は大雑把に２種類の段階を踏む,第一段階ではrawコードの解析(パース)を行う、そして第二段階で、実行を行う。
    • メタプログラミングを使うことで、第一段階が終わった段階のコードに対してコードの変換・生成を行うことができる。
    • それにより、ネイティブなjuliaのコードと同じ速度で実行が可能となっている、そのために、マクロは必要なのだ。 たぶん

ちょっと変わった型システム

  • juliaの型(クラス)システムは、多重ディスパッチと呼ばれるものを利用し多態性を表現
    • オブジェクト指向言語ではクラス(型）にメソッド(関数)が所属
    • juliaでは、関数(メソッド)に型(クラス)が所属
      • 具体的には、関数の引数の型によって、関数の挙動が変化
        • 関数のオーバーロードに近い形のものを想像してくださいです

型

型は、大きく分けると

  • 抽象型
  • 具象型

に分けることができます。

  • 抽象型は、変数を持つことができません!!! さらに、インスタンスも生成できません。
  • 具象型は　継承できません!!

従って、移譲(Delegation)を使い継承関係を構築することもできます。

どうしても使いたい場合は、trait-mixin マクロなどを作ったりすることもできます。

type Quaternion{T<:Real} <:Number
    x::T
    y::T
    z::T
    w::T
    #内部にもコンストラクタ(型と同一名の関数)を定義できます。詳しくは 公式サイトで確認してください。
end

Quaternion() = Quaternion(0,0,0,0)

+(q1::Quaternion, q2::Quaternion) = Quaternion(q1.x + q2.x, q1.y + q2.y, q1.z + q2.z, q1.w + q2.w)
-(q1::Quaternion, q2::Quaternion) = Quaternion(q1.x - q2.x, q1.y - q2.y, q1.z - q2.z, q1.w - q2.w)
*(q1::Quaternion, q2::Quaternion) = Quaternion(q1.w * q2.w - q1.x * q2.x - q1.y * q2.y - q1.z * q2.z,
                                               q1.y * q2.z - q1.z * q2.y + q1.w * q2.x + q1.x * q2.w,
                                               q1.z * q2.x - q1.x * q2.z + q1.w * q2.y + q1.y * q2.w,
                                               q1.x * q2.y - q1.y * q2.x + q1.w * q2.z + q1.z * q2.w)
*(q1::Quaternion, r::Real) = Quaternion(q1.x*r, q1.y*r, q1.z*r, q1.w*r)
*(r::Real, q1::Quaternion) = *(q1,r)

Conversion and Promotion

juliaでは上位型への暗黙的な型キャストは行いません。
なので、型が適用できなかった場合に明示的に型変換を行う関数promoteとそれに関連したconvert関数が必要になってきます。

  • Promotion 上位の型への変換 ->
    • promote_rule関数によって型の変換関係が定義されています。
    • promote_type関数がpromote_ruleに基づく型の変換を行っています。
  • Conversion
    • convert 関数 promote 関数が型変換を行うときに用いる関数。
    • 型変換の定義を行う場合は convert 関数に定義してください。

実際の処理の流れ

  1. 型Tと型Sの間で何かしら計算出来ない場合はfallbackメソッドが呼ばれる
  2. fallbackメソッドでpromote関数が呼ばれる
  3. promote関数ではconvert(promote_type(T,S),x)が呼ばれる
  4. ここでpromote_type(T,S)の返り値はpromote_ruleによって決定され、より大きな共通の型を返す
  5. 返された型にxの値を変換
