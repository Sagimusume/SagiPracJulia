Task(コルーチン)

コルーチンとは,処理をいったん中断し,その後必要に応じてにその処理の続きから再開することができる制御構造です

  • produce
    処理を中断する(指定する)

  • Task
    タスクを生成する

  • consum
    タスクを再開する

パラレル化する際にも似た様な制御構造を利用しています

In [153]: 
function producer()
    produce("start")
    for n=1:4
        produce(2n)
    end
    produce("stop")
end;

p = Task(producer);

#条件付き分布から一つサンプリング
function sample_one(x)
    new_x = rand(Normal(ρ*x[2], 1-ρ^2))
    new_y = rand(Normal(ρ*new_x, 1-ρ^2))
    return [new_x, new_y]
end

#イテレーション回数
N = 600
BURNIN = 0

#コルーチン
function sampler(BURNIN)
  x = zeros(2)
  for i = 1:BURNIN
    x = sample_one(x)
  end
  while true
    x = sample_one(x)
        produce(x)   #produce
  end
end
  
xs, ys = zeros(N),zeros(N)

#コルーチン作成
sim = @task sampler(BURNIN)

#ループで任意回数サンプルします
for i = 1:N
  xs[i],ys[i] = consume(sim)
end

juliaにおけるコードの評価

juliaにおけるコード評価は大雑把に２種類の段階を踏む,第一段階ではrawコードの解析(パース)を行う、そして第二段階で、実行を行う。
メタプログラミングを使うことで、第一段階が終わった段階のコードに対してコードの変換・生成を行うことがでくる。
それにより、ネイティブなjuliaのコードと同じ速度で実行が可能となっている
そのために、マクロは必要なのだ。 たぶん

ちょっと変わった型システム

juliaの型(クラス)システムは、多重ディスパッチと呼ばれるものを利用し(多態性)表しています。

pythonでは、クラス(型）にメソッド(関数)が属していましたが 。
juliaでは、関数(メソッド)に型(クラス)が属します。

具体的には、関数の引数の型によって、関数の挙動が変化します。

関数のオーバーロード、に近い形ものを想像してくださいです。

型

型は、大きく分けると * 抽象型
* 具象型

に分けることができます。

  • 抽象型は、変数を持つことができません!!! さらに、インスタンスも生成できません。
  • 具象型は　継承できません!!

従って、移譲(Delegation)を使い継承関係を構築することもできます。

どうしても使いたい場合は、trait-mixin マクロなどを作ったりすることもできます。 ____

                                               q1.z * q2.x - q1.x * q2.z + q1.w * q2.y + q1.y * q2.w,
                                               q1.x * q2.y - q1.y * q2.x + q1.w * q2.z + q1.z * q2.w)
*(q1::Quaternion, r::Real) = Quaternion(q1.x*r, q1.y*r, q1.z*r, q1.w*r)

Conversion and Promotion

juliaでは上位型への暗黙的な型キャストは行いません。

  • Conversion
    convert 関数 promote 関数が型変換を行うときに用いる関数。 型変換の定義を行う場合は convert 関数に定義してください。
